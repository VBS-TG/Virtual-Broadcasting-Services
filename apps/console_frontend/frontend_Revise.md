# 前端增補需求（Console UI）

> **範圍**：僅 `apps/console_frontend` 的畫面、互動、狀態、文案與導覽；不含後端實作、OpenAPI 契約或非前端的設定檔。

---

## 產品定位（導播後台，非開發後台）

- **正式 Console = 決策中心**，不是終端機：操作員若要查 Socket 封包、記憶體或逐行報錯，應使用 **CLI／維運工具**，不在此處堆砌除錯 UI。
- **介面語彙**：以「狀態 Status」取代「Debug」——使用者看到的是 **可做什麼決策／目前是否安全／下一步建議**，而不是 HTTP 狀態碼與原始錯誤正文。
- **資訊架構（三大區）**：
  | 區塊 | 角色 | 說明 |
  | --- | --- | --- |
  | **戰術操作 (Live)** | 直播當下 | Dashboard、Switcher、Telemetry 等——低延遲、少打字、少確認步驟。 |
  | **編排設定 (Setup)** | 開播前／彩排 | Pipeline（鏈路與映射）、Runtime、版本與套用——可較完整，但仍避免工程式表單。 |
  | **環境狀態 (Health)** | 全程 | 語意化連線／認證／延遲狀態——**不**把底層錯誤碼當主標題。 |

---

## Pipeline 與 Show Config：整併（勿拆成兩套互斥功能）

**原則**：同一套設定資料只有 **一份真相**；分成兩個路由是 **流程分工**，不是兩份不相干的設定。

| 路由（建議） | 定位 | 使用者心智 |
| --- | --- | --- |
| **`/pipeline`**（或 Production／製作） | **編輯態（Draft）** | 調整鏈路品質預設、Mapping、來源標籤、Multiview——「我在編輯這一場的製作參數」。 |
| **`/show-config`**（Show Control／演出控制） | **版本與套用（Apply）** | 檢視草稿／已生效版本、歷史、執行套用／回滾——「我把編好的東西發布到哪一版」。 |

- **禁止**：在正式流程提供 **Raw JSON 編輯區** 作為主要編輯方式（若有「進階／匯出」可另議，且不列入操作員主路徑）。
- **取代**：表單、預設集、畫布式配置（見下節）。

---

## 1. Pipeline（`/pipeline`）— 製作／鏈路（高優先）

### 1.1 整線畫質：品質預設集（非參數手動）

- **不要**：要求使用者理解 H.265 Profile、具體 GOP 等工程參數。
- **要**：**品質預設集（Quality Presets）**，例如：**高畫質**、**低延遲**、**行動／平衡**。選一個詞彙即能對應底層映射（對應關係可由產品／後端定義，前端呈現為 **卡片或 Segmented Control**）。
- **端到端語意**：仍以 **Capture → Route → Engine** 呈現（步驟條或三欄），讓製作人知道影響範圍是「整條鏈」，不是單一機器上的檔案。
- **與 Runtime 分工**：Runtime 頁維持 **路數、SRT URI、aux 等**；Pipeline 維持 **格式／品質語意與映射**，側欄分開入口。

### 1.2 按鈕 Mapping — 視覺畫板優先

- **優先**：**畫板／格位模式（Visual Canvas）**——點選一個「槽位／按鈕區」→ 彈出選項選 **訊號／來源**（比純列表直覺）。
- **備援**：較密的列表編輯可作次要視圖（例如「列表檢視」折疊區）。
- **重置為預設**、變更前 **摘要確認**（與套用流程共用互動 pattern）。

### 1.3 來源標籤：顯示名 × 短代號

- **顯示名**：給 Multiview／監看標題等較長展示用。
- **短代號（例如 4 字以內）**：給 Switcher 按鈕、窄欄位——避免長字被截斷後無法辨識。
- **UX**：兩欄分開編輯；過長 **Tooltip** 顯示全名。

### 1.4 Multiview Mapping — 與畫板一致

- **格位 ↔ 來源**：與 1.2 同一套「點格→選來源」邏輯；與 **Popout Multiviewer** 必須 **同源設定**（避免主控與浮窗不一致）。

### 1.5 Operator 唯讀例外

- **方案 B（隱藏無權限選單）** 為主（見 §5）；**例外**：`/pipeline` 允許 Operator **進入但唯讀**，使其在直播壓力下仍能 **確認目前按鈕／格位對應**，但不可改 Draft。

---

## 2. Show Control（`/show-config`）— 版本與套用（高優先）

- **草稿／已生效／時間戳**：以 **版本卡／時間軸** 呈現，非 JSON。
- **套用／回滾**：前置 **摘要確認**；結果以 **節點維度成功／失敗／略過** 呈現（見 §3），**不**以整段 log 為主視覺。
- **歷史**：版本、套用時間、摘要狀態（成功／部分失敗）。

---

## 3. Runtime 套用結果：進度與語意，非工程日誌（中優先）

- **套用進行中**：**階段式進度（Step Indicator）**，例如：**驗證 → Route → Engine → 完成**（依實際下游可調整步驟名）。
- **失敗**：該步驟 **標紅**，主文案用 **導播可理解的短句**（例如「Route 節點拒絕套用：格式不支援」）；**技術代碼／片段**預設摺疊，放在 **「檢視原因」** 內展開。
- **Dashboard**：與 Runtime **同一資料來源** 顯示「上次套用結果摘要」，避免兩處矛盾。

---

## 4. 系統健康（`/system`）：語意化狀態（中優先）

- **不要**（主畫面）：`Error 401`、`Socket Timeout`、原始 stack 等當 **第一層標題**。
- **要**（對應語意）：
  - **通訊受阻**（對應 401／403）：說明「認證失效或權限受限」，並提供 **一鍵重新登入／重新校驗**（若產品允許）。
  - **回應遲緩**：說明「節點反應緩慢，可能網路負載過高」（可搭配延遲閾值，純前端展示策略亦可）。
  - **連線中斷**：說明「無法連上目標，請檢查網路或隧道」——**不**假設使用者懂 DNS／Tunnel 細節。
- **除錯資訊**：詳細代碼、trace_id、原始狀態碼等，放在 **「複製診斷資訊」** 按鈕（複製到剪貼簿給工程師），**不**預設鋪滿畫面。
- **Settings**：仍保留 **Route Base URL**（供健康檢查與描述用），標籤維持 **使用者友善**。
- **檢查目標**：Console、Engine、Route **三列**（與 Telemetry 節點狀態 **色票語彙一致**，避免同一節點多種矛盾狀態）。

---

## 5. 角色與導覽（採方案 B + Pipeline 唯讀例外）（中優先）

- **Operator**：側欄 **不顯示** `Settings`、`RentalSessions` 等管理項，減少誤觸與心理負載。
- **Admin**：完整選單。
- **例外**：`/pipeline` **Operator 可進入唯讀**（見 §1.5）。
- **一致性**：`Sidebar` 與 `BottomNav` 規則相同。

---

## 6. 安全栓（Safety Pin）— 正式環境保護（高優先／與 Live 流程綁定）

### 6.1 Live 模式（正式模式開關）

- **位置建議**：頁首／右上角 **Live Mode** 開關（文案可依設計微調）。
- **Live 開啟時**：所有會 **中斷訊號／大範圍改設定** 的動作 **上鎖（Disabled）**：包含 **Apply、Rollback**、以及產品定義之「會導致斷流」的 Pipeline／Runtime 變更。
- **若要操作**：需先 **關閉 Live 模式**（或具同等語意的「允許變更設定」流程）。

**是否值得做**：對 **正式播出防誤觸** 很有幫助；需注意 **彩排／緊急 hotfix** 時多一步切換可能造成摩擦——可輔以 **簡短確認 Modal**（「確定在目前直播中開放設定變更？」）作為 Live 關閉時的二次確認，由產品決策。

### 6.2 重大操作：長按（Dead-man's）

- **Rollback**（及產品定義之其他重大動作）：需 **長按約 2 秒** 才觸發，避免誤觸。
- **一般 Apply**：可用一般確認對話；若與 Rollback 同級風險，可再升級為長按。

---

## 7. 設定頁（Settings）（低優先）

- **Route Base URL**、API／Engine URL、逾時、Telemetry 間隔等——**維持簡潔**，不混入金鑰或 Tunnel 秘訣。
- **輪詢**：套用狀態輪詢頻率可獨立於 Telemetry，避免過載。

---

## 8. 操作日誌（`/logs`）（低優先）

- 若以「製作人」視角：日誌可偏向 **誰／何時／做了什麼決策**，而非原始 request dump。
- 進階篩選（套用／登入等）預留即可。

---

## 9. 無障礙與測試（橫向）

- 長流程：**Loading**、防重複送出、鍵盤可操作。
- 保留 **`data-testid`** 於關鍵步驟（套用、Live 開關、長按鈕）。

---

## 側欄資訊架構（建議標籤）

- **Production（製作）**：進入 **`/pipeline`**（鏈路品質預設、Mapping、標籤、Multiview）。
- **Show Control（演出控制）**：進入 **`/show-config`**（版本、套用、回滾、歷史）。
- 既有：Dashboard、Switcher、Runtime、Telemetry、System、Logs；Settings／Rentals 僅 Admin。

---

## 建議實作順序

1. **IA + Live／長按規格**：側欄分組、Live 鎖定範圍、Rollback 長按。  
2. **`/pipeline`**：預設集、畫板式 Mapping、顯示名／短代號、Multiview（無 Raw JSON 主路徑）。  
3. **`/show-config`**：版本視圖、套用／回滾、結果用 **進度／語意化** 呈現。  
4. **`/system`**：語意狀態 + **複製診斷資訊** + Route 第三檢查目標。  
5. **Runtime／Dashboard**：與 §3 同一套用摘要元件。  
6. **角色**：方案 B + Pipeline Operator 唯讀。

---

## 刻意不包含（提醒）

- Access Service Token、JWT、Tunnel **金鑰**：非前端需求。  
- **Capture** 無 health URL 時可不強制顯示該列，直至可檢查為止。
